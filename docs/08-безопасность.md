# Рекомендации по безопасности

## API Key

- **НИКОГДА** не передавайте API Key в публичные репозитории
- Храните в `~/.aapanel/servers.conf` — файл доступен только вашему пользователю
- Периодически ротируйте ключ в панели aaPanel
- Используйте переменные окружения при необходимости: `export AAPANEL_API_KEY=...`

## IP Whitelist

- Держите в whitelist только необходимые IP
- Регулярно удаляйте устаревшие IP
- По возможности используйте VPN с фиксированным IP

## Firewall

- Держите firewall **всегда активным**
- Открывайте только необходимые порты
- По возможности ограничивайте правила конкретными IP
- Закрывайте порты разработки после тестирования

```bash
# Проверить открытые порты
aapanel firewall GetList '{"p":1,"limit":100}'

# Закрыть ненужный порт
aapanel firewall DelAcceptPort '{"id":RULE_ID}'
```

## SSH

- Используйте SSH ключи вместо паролей
- Смените стандартный порт SSH
- Отключите вход под root

```bash
# Проверить конфигурацию SSH
aapanel safe_ssh GetSshInfo
```

## SSL/HTTPS

- Всегда используйте SSL на production сайтах
- Обновляйте сертификаты до истечения срока
- Настройте принудительный HTTPS (редирект HTTP → HTTPS)

## Базы данных

- Используйте надёжные пароли (минимум 16 символов, разные типы)
- Ограничивайте доступ конкретными IP (избегайте `%`)
- Делайте регулярные резервные копии

```bash
# Ограничить доступ IP приложения
aapanel database SetDatabaseAccess '{"name":"mydb","access":"127.0.0.1"}'
```

## Резервные копии

- Настройте автоматические ежедневные бэкапы
- Периодически тестируйте восстановление
- Храните бэкапы в месте, отличном от основного сервера

```bash
# Автоматический ежедневный бэкап базы данных
aapanel crontab AddCrontab '{
  "name": "Daily DB Backup",
  "type": "day",
  "hour": "3",
  "minute": "0",
  "sType": "database",
  "sName": "mydb",
  "backupTo": "localhost",
  "save": "7"
}'

# Автоматический ежедневный бэкап сайта
aapanel crontab AddCrontab '{
  "name": "Daily Site Backup",
  "type": "day",
  "hour": "3",
  "minute": "30",
  "sType": "site",
  "sName": "mysite.com",
  "backupTo": "localhost",
  "save": "7"
}'
```

## Обновления

- Держите aaPanel обновлённым
- Регулярно обновляйте: Node.js, Nginx, MySQL, PHP

```bash
# Проверить обновление панели
aapanel ajax UpdatePanel '{"check":true}'
```

## Мониторинг

- Настройте алерты при высокой нагрузке CPU/памяти
- Регулярно просматривайте логи ошибок
- Используйте cron jobs для health checks

```bash
# Автоматический health check
aapanel crontab AddCrontab '{
  "name": "Health Check",
  "type": "hour",
  "hour": "",
  "minute": "0",
  "sBody": "curl -sf http://localhost:3000/health || echo \"APP DOWN\" | mail -s \"Alert\" your@email.com",
  "sType": "toShell"
}'
```
